## 1\. 项目概述

### 1.1 背景与目标

随着家庭中闲置数码设备（旧手机、旧平板、旧笔记本）越来越多，且二手残值较低，本项目旨在将这些设备变废为宝。通过构建一套统一的家庭数字标牌系统，利用闲置设备的屏幕和计算能力，在家庭各个角落提供动态、可远程控制的信息展示服务，并具备主动提醒能力，如定时事项提醒和紧急安全告警。

### 1.2 目标用户

*   **家庭管理员**：负责配置和发布显示内容（如夫妻双方、成年子女）。
    
*   **家庭成员**：观看显示屏信息的用户（老人、孩子、访客）。
    
*   **外部系统**：家庭自动化系统（如HomeAssistant）、备忘录应用（如Notion、飞书文档）、安防系统，通过API与本系统集成。
    

### 1.3 范围

本系统包含服务端管理后台与客户端显示程序。服务端基于**Node.js + Express**框架，运行在家庭内部的NAS、常开PC或树莓派上；客户端运行在各种闲置设备的浏览器中。

## 2\. 总体描述

### 2.1 产品视角

本系统是一个基于 **Node.js后端 + 浏览器前端** 的轻量级信息发布与主动提醒系统。核心是一个基于Express框架的中央服务器，负责聚合数据并提供RESTful API；客户端设备通过浏览器全屏显示内容，并具备音频播放能力，可响应定时与紧急提醒。

### 2.2 关键功能概要

*   **集中管理**：通过一个网页后台，统一管理所有房间/设备的显示内容。
    
*   **纯Web客户端**：所有显示设备均通过浏览器访问，无需开发专用客户端应用。
    
*   **丰富的内置组件**：支持时间、天气、图片轮播、Markdown文本（备忘录）、RSS订阅等。
    
*   **多画面轮播**：单个屏幕支持配置多个画面，每个画面可包含不同的组件布局。系统支持按设定的时间间隔和顺序，以淡入淡出效果自动轮播画面。
    
*   **定时提示**：支持在指定的时间段内，在屏幕上高亮显示提示信息并播放提示音，适用于出门提醒、服药提醒等场景。
    
*   **紧急提示**：支持最高优先级的紧急消息推送，可立即中断所有常规显示和定时提示，全屏高亮展示紧急信息并循环播放警示音，直至手动确认解除。
    
*   **开放式API**：提供完整的RESTful API，允许外部程序动态修改显示内容、创建定时提示和触发紧急提示，实现与家庭智能系统的深度集成。
    

### 2.3 运行环境

*   **服务器端**：
    
    *   操作系统：Windows 10/11, Linux (Ubuntu/Debian), macOS, Docker 容器。
        
    *   运行时：Node.js (v16+)
        
    *   框架：Express
        
    *   硬件：任何支持Node.js运行的设备（NAS、低功耗PC、树莓派）。
        
*   **客户端**：
    
    *   任何带有现代浏览器（Chrome 80+ / Safari 13+）的设备，如 iPhone、iPad、MacBook、Android平板。
        
    *   浏览器需支持：Fullscreen API、WebSocket、Web Audio API。
        

## 3\. 具体需求

### 3.1 功能需求

#### 3.1.1 管理后台 (Web端)

| ID  | 模块  | 需求描述 | 优先级 |
| --- | --- | --- | --- |
| F001 | 设备管理 | 管理员可以添加新设备（生成唯一ID或访问链接），并查看设备最后在线状态、浏览器信息。 | 高   |
| F002 | 布局组件库 | 提供以下基础组件库：  <br>\- **时钟**：数字/模拟样式，可配置时区。  <br>\- **天气**：根据配置的城市显示实时天气及未来预报（需接入免费天气API）。  <br>\- **文本**：支持富文本编辑，用于显示备忘录、注意事项。  <br>\- **图片**：支持上传图片或指定网络图片URL，支持轮播间隔设置。  <br>\- **Iframe/网页嵌入**：嵌入特定网页（如新闻网站、股票走势）。 | 高   |
| F003 | 画面管理 | 管理员可以创建多个“画面”（Scene）。每个画面是一个独立的布局（包含上述组件的组合）。 | 高   |
| F004 | 多画面轮播配置 | 针对单个设备，支持分配多个画面。管理员可配置：  <br>\- 启用/禁用某个画面。  <br>\- 设置每个画面的显示时长（如画面A显示10秒，画面B显示20秒）。  <br>\- 预览轮播顺序。  <br>\- 系统在客户端需实现**淡入淡出**转场效果。 | 高   |
| F005 | 定时提示管理 | 管理员可以创建定时提示任务，配置以下参数：  <br>\- **生效时间段**：开始时间与结束时间（支持重复规则，如每天、工作日）。  <br>\- **提示设备**：指定单个或多个目标设备。  <br>\- **提示内容**：文本内容，支持简单格式（如字体大小、颜色）。  <br>\- **提示样式**：高亮模式（如闪烁、背景色反转）。  <br>\- **提示音**：选择内置提示音或上传音频文件。  <br>\- **优先级**：在同时有多个定时提示时，按优先级显示。 | 高   |
| F006 | 紧急提示管理 | 管理员可以手动触发紧急提示，配置以下参数：  <br>\- **提示设备**：指定单个或多个目标设备。  <br>\- **提示内容**：文本内容。  <br>\- **提示样式**：强制全屏显示，最高亮度，高对比度背景色（如红色），文字可闪烁。  <br>\- **提示音**：选择内置紧急警示音或上传音频文件，支持循环播放。  <br>\- **确认机制**：紧急提示持续显示，直到管理员在后台或通过API手动解除。 | 高   |
| F007 | 内容分发 | 支持将配置好的画面组和轮播规则分配给特定的设备或设备组（如“厨房组”、“书房组”）。 | 高   |

#### 3.1.2 客户端显示

| ID  | 模块  | 需求描述 | 优先级 |
| --- | --- | --- | --- |
| C001 | 自动启动 | 设备开机后通过浏览器访问指定URL，自动全屏进入显示模式，无操作界面干扰（Kiosk Mode）。 | 高   |
| C002 | 数据拉取 | 客户端定期（如每5分钟）向服务器请求最新的显示内容、画面列表、轮播配置、定时提示任务及紧急提示状态。 | 高   |
| C003 | 画面轮播引擎 | 客户端根据服务器下发的配置，按顺序加载不同画面，并实现**淡入淡出**效果的无缝切换。计时器需精准，切换过程不卡顿。 | 高   |
| C004 | 优先级管理引擎 | 客户端需实现三层优先级体系：  <br>1\. **最高级**：紧急提示 - 无条件中断一切显示，全屏展示紧急内容并播放声音。  <br>2\. **中级**：定时提示 - 在当前画面之上叠加显示高亮提示条/浮层，并播放提示音（不影响轮播后台运行）。  <br>3\. **常规级**：画面轮播。 | 高   |
| C005 | 定时提示渲染 | 在定时提示生效时段内，按配置的样式在屏幕上渲染提示信息。支持：  <br>\- 顶部/底部固定条。  <br>\- 屏幕中央浮动窗口。  <br>\- 文字闪烁、背景高亮等效果。  <br>\- 提示音播放（需处理浏览器自动播放策略）。 | 高   |
| C006 | 紧急提示渲染 | 接收到紧急提示后：  <br>\- 立即暂停所有画面轮播和定时提示。  <br>\- 全屏显示紧急内容，背景色为醒目的红色或黄色。  <br>\- 循环播放紧急提示音。  <br>\- 提供“已确认”按钮（或在后台远程解除后自动恢复）。 | 高   |
| C007 | 断网显示 | 当网络断开时，客户端至少能显示本地缓存的上一次内容，或者显示默认的离线画面。定时提示和紧急提示在断网时不可用。 | 中   |
| C008 | 音频管理 | 支持同时管理多个音频上下文，确保提示音可正常播放，且紧急提示音可打断定时提示音。 | 中   |

### 3.2 RESTful API 需求

为了让外部系统能够动态控制显示内容，系统需提供基于Node.js + Express的RESTful API。

| ID  | 端点  | 方法  | 描述  | 请求体示例 | 优先级 |
| --- | --- | --- | --- | --- | --- |
| API001 | `/api/devices` | GET | 获取所有设备列表及其状态 | \-  | 中   |
| API002 | `/api/devices/:deviceId/config` | GET | 获取指定设备的完整配置 | \-  | 高   |
| API003 | `/api/devices/:deviceId/scenes` | PUT | 批量更新指定设备的画面列表及轮播配置 | `{ "scenes": [{"id": "s1", "duration": 10}], "transition": "fade" }` | 高   |
| API004 | `/api/devices/:deviceId/active-scene` | POST | 强制切换当前显示的画面 | `{ "sceneId": "s2" }` | 高   |
| API005 | `/api/scenes/:sceneId/content/text` | PATCH | 动态更新指定画面中的文本组件内容 | `{ "componentId": "memo_1", "content": "晚上8点开会" }` | 高   |
| API006 | `/api/reminders/timed` | POST | 创建定时提示任务 | `{ "deviceIds": ["kitchen"], "startTime": "2023-10-27T14:45:00Z", "endTime": "2023-10-27T15:15:00Z", "content": "带钱包出门", "style": "blink", "sound": "bell.mp3", "repeat": "daily" }` | 高   |
| API006b | `/api/reminders/timed/:id` | DELETE | 取消定时提示任务 | \-  | 高   |
| API007 | `/api/reminders/emergency` | POST | 触发紧急提示 | `{ "deviceIds": ["all"], "content": "疑似煤气泄漏，请检查！", "sound": "alarm.mp3" }` | 高   |
| API007b | `/api/reminders/emergency/:id` | DELETE | 解除紧急提示 | \-  | 高   |
| API008 | `/api/reminders/emergency/status` | GET | 查询当前是否有未解除的紧急提示 | \-  | 中   |

_注：所有API需进行简单的身份验证，如Header中包含预共享的API Key。_

### 3.3 非功能需求

| ID  | 类型  | 需求描述 | 优先级 |
| --- | --- | --- | --- |
| N001 | 易用性 | 服务器端部署应尽量简单，提供一键Docker启动脚本。客户端应简化配置，通过扫码或输入URL即可绑定。 | 高   |
| N002 | 资源占用 | 客户端占用CPU和内存尽可能低，主要以静态页面为主，避免复杂的动画导致老旧设备发热。 | 高   |
| N003 | 可靠性 | 服务端需具备自动恢复能力，客户端长时间运行不应崩溃或内存泄漏。紧急提示模块需独立高可用。 | 高   |
| N004 | 实时性 | 紧急提示从API触发到客户端响应的延迟应小于2秒。定时提示的触发时间误差应小于10秒。 | 高   |
| N005 | 兼容性 | 网页客户端必须兼容 iOS Safari、macOS Safari 和 Chrome 的最新版本及前两个主要版本。音频播放需处理移动端限制。 | 高   |
| N006 | 安全性 | 管理后台需设置密码。API需支持API Key验证，防止未授权访问。紧急提示API需额外加强验证。 | 高   |

## 4\. 数据交互与界面原型描述

### 4.1 主要流程

1.  **管理员部署**：在NAS上通过Docker运行基于Node.js的服务器镜像。
    
2.  **管理员配置**：登录后台，创建“家庭备忘录”画面和“时钟天气”画面。在“客厅”设备设置中，将这两个画面都分配给它，并设置各显示15秒，开启淡入淡出。
    
3.  **设备绑定**：闲置iPad访问专用URL，进入全屏模式，开始轮播画面。
    
4.  **定时提醒场景**：用户通过手机上的家庭自动化App，调用API006创建一个定时提示：“下午2:45-3:15，客厅屏幕显示‘带钱包出门’，闪烁，播放提示音”。到时间后，iPad在轮播画面顶部叠加显示闪烁的提醒条，并发出提示音。
    
5.  **紧急提醒场景**：厨房煤气传感器触发泄漏告警。家庭安防系统调用API007，向所有设备推送紧急提示：“疑似煤气泄漏！”。所有屏幕立即变为红色全屏，显示警告文字，并循环播放刺耳警报声。用户处理完毕后，在后台点击解除，屏幕恢复正常轮播。
    
6.  **外部API集成**：家庭成员在第三方备忘录App中修改了购物清单。家庭自动化系统通过API005动态更新“家庭备忘录”画面中的文本内容。
    

### 4.2 界面原型文字描述

*   **管理后台**：
    
    *   左侧为导航栏（仪表盘、设备列表、画面设计器、定时提示、紧急提示、系统设置）。
        
    *   画面设计页为拖拽式界面，中间是手机/平板/电脑的模拟器画布，右侧是组件选择面板和属性设置。
        
    *   紧急提示控制台配有醒目的红色触发按钮。
        
*   **客户端显示**：
    
    *   全屏显示，无地址栏和状态栏（通过浏览器全屏API实现）。
        
    *   背景色为深色或黑色，信息显示清晰，对比度高，适合长时间低功耗显示。
        
    *   定时提示以浮层形式出现在画面之上，不影响背景轮播。
        
    *   紧急提示全屏覆盖，背景红色，文字白色且闪烁。
